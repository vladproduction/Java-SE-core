Understanding HashSet Internals

1. Internal Structure

HashSet is backed by a HashMap where elements are stored as keys with a dummy value.

  • Hashing: Uses hash codes to determine element storage location.
  • No Duplicates: Only stores unique elements, ignoring duplicates.
  • No Order: Does not maintain insertion order of elements.
  • Null Element: Allows one null element.
2. Time Complexity

HashSet operations have constant time complexity on average:

  • add(element): O(1)
  • remove(element): O(1)
  • contains(element): O(1)
  • size(): O(1)
3. Example Code

import java.util.HashSet;
import java.util.Iterator;

public class HashSetExample {
    public static void main(String[] args) {
        HashSet colors = new HashSet<>();

        // Adding elements
        colors.add("Red");
        colors.add("Green");
        colors.add("Blue");
        colors.add("Red");  // Duplicate - will be ignored

        // Display elements
        System.out.println("Colors: " + colors);

        // Check size
        System.out.println("Number of colors: " + colors.size());

        // Check if contains element
        System.out.println("Contains Green? " + colors.contains("Green"));
        System.out.println("Contains Yellow? " + colors.contains("Yellow"));

        // Remove element
        colors.remove("Green");

        // Iterate using Iterator
        System.out.println("Colors after removal:");
        Iterator< String> iterator = colors.iterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }

        // Using for-each loop
        System.out.println("Using for-each:");
        for (String color : colors) {
            System.out.println(color);
        }

        // Clear all elements
        colors.clear();
        System.out.println("Is empty? " + colors.isEmpty());
    }
}
Conclusion

HashSet is the go-to collection when you need to ensure uniqueness of elements and require fast access, addition, and removal operations. It's ideal for scenarios where order doesn't matter but uniqueness does, such as removing duplicates from a collection.
