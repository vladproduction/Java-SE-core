Understanding HashMap Internals

1. Internal Structure

HashMap uses an array of buckets to store entries. Each bucket can hold multiple entries that collide at the same index.

  • Buckets: Array structure that holds linked lists or trees for managing entries.
  • Hashing Mechanism: Uses an efficient hash function to determine the index.
  • Collisions: Handled by linked lists before Java 8, and by balanced trees afterwards.
  • Load Factor and Resizing: HashMap has a load factor (default is 0.75) that determines when the map should resize. When the number of entries exceeds capacity * load factor, the map will grow, and rehashing will occur to redistribute the entries.
2. Time Complexity

The average time complexity for operations in a HashMap is O(1).

  • Put: O(1)
  • Get: O(1)
  • Remove: O(1)
3. Example Code

import java.util.HashMap;

public class HashMapExample {
    public static void main(String[] args) {
        HashMap< String, Integer> map = new HashMap<>();
        map.put("Apple", 1);
        map.put("Banana", 2);
        map.put("Orange", 3);

        System.out.println("Apple: " + map.get("Apple"));
        System.out.println("Banana: " + map.get("Banana"));

        System.out.println("Contains Orange? " + map.containsKey("Orange"));
        map.remove("Banana");
        System.out.println("Contains Banana? " + map.containsKey("Banana"));

        for (String key : map.keySet()) {
            System.out.println(key + ": " + map.get(key));
        }
    }
}
Conclusion

HashMaps are powerful structures for key-value storage and retrieval. Understanding their mechanics helps in efficient programming.
