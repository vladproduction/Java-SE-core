Understanding LinkedList Internals

1. Internal Structure

LinkedList implements a doubly-linked list data structure, where each element (node) contains references to both previous and next nodes.

  • Nodes: Each element is stored in a node containing the data and references to adjacent nodes.
  • Head and Tail: Maintains references to both the first and last elements for efficient access.
  • Bi-directional: Can be traversed in both directions (forward and backward).
2. Time Complexity

LinkedList operations have varying time complexities:

  • get(index): O(n)
  • add(element): O(1)
  • add(index, element): O(n)
  • remove(element): O(n)
  • addFirst/addLast: O(1)
  • removeFirst/removeLast: O(1)
3. Example Code

import java.util.LinkedList;

public class LinkedListExample {
    public static void main(String[] args) {
        LinkedList animals = new LinkedList<>();

        // Adding elements
        animals.add("Dog");
        animals.add("Cat");

        // Adding at the beginning and end
        animals.addFirst("Lion");
        animals.addLast("Elephant");

        // Display all elements
        System.out.println("Animals: " + animals);

        // Get first and last elements
        System.out.println("First animal: " + animals.getFirst());
        System.out.println("Last animal: " + animals.getLast());

        // Adding at specific position
        animals.add(2, "Tiger");

        // Iterating using for-each
        System.out.println("All animals:");
        for (String animal : animals) {
            System.out.println(animal);
        }

        // Queue operations
        animals.offer("Giraffe");  // Adds to end
        System.out.println("Peek: " + animals.peek());  // View first
        System.out.println("Poll: " + animals.poll());  // Remove first

        // Stack operations
        animals.push("Monkey");  // Add to front
        System.out.println("Pop: " + animals.pop());  // Remove from front
    }
}
Conclusion

LinkedList is ideal for frequent insertions and deletions, especially at the beginning or end of the list. It also implements both Queue and Deque interfaces, making it versatile for different use cases. However, it's less efficient for random access operations compared to ArrayList.
