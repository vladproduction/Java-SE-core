Abstraction in Java

Abstraction is one of the fundamental concepts of Object-Oriented Programming in Java. It refers to the process of hiding the implementation details and showing only the essential features to the user. Abstraction helps in reducing complexity and allows the programmer to focus on what an object does instead of how it does it.

Types of Abstraction in Java

1. Data Abstraction

Data abstraction is achieved using abstract classes and interfaces. It helps in exposing only necessary parts of the data to the user and hiding the implementation details.

Interfaces in Java

  • An interface is a blueprint of a class.
  • Interfaces contain final and static variables (constants).
  • Interfaces primarily contain abstract methods (unimplemented methods). From Java 8 onwards, default and static methods are also allowed.
  • An abstract method has a method signature but no body.
  • All methods in an interface are public by default.
  • Interfaces support multiple inheritance.
  • An interface is defined using the interface keyword.
  • A class extends another class, an interface extends another interface, but a class implements an interface.
  • We can create an object reference for an interface, but we cannot instantiate an interface directly.

Abstract Classes

An abstract class can have both abstract and concrete methods. It provides a partial implementation and is meant to be extended by subclasses that complete the missing functionality.

Access Modifiers in Java

  • public - The variable or method is accessible from anywhere.
  • protected - Accessible within the same package and by subclasses (even in different packages).
  • default (no modifier) - Accessible only within the same package.
  • private - Accessible only within the same class.

Example

interface Vehicle {
    void start();
    void stop();
}

class Car implements Vehicle {
    public void start() {
        System.out.println("Car started");
    }

    public void stop() {
        System.out.println("Car stopped");
    }
}

public class Main {
    public static void main(String[] args) {
        Vehicle myCar = new Car();
        myCar.start();
        myCar.stop();
    }
}

This example demonstrates how abstraction hides the actual implementation of start and stop methods inside the Car class and exposes only the method signatures through the Vehicle interface.
